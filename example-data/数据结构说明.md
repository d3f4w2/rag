# 示例数据结构说明（`example_data`）

## 1. 数据集总览

- 根目录包含 2 个类别目录：`HSIL/`、`LSIL/`。
- 每个类别目录下各有 2 个受检者目录（共 4 人）。
- 每个受检者目录均包含固定 7 个原始文件：
  - `1.jpg` ~ `5.jpg`（共 5 张阴道镜图像）
  - `姓名_检查报告_时间戳.pdf`（1 份检查报告）
  - `着色情况.txt`（1 份结构化文字描述）
- 原始样本总计 28 个文件，约 6.10 MB（`6,395,159` 字节）。

## 3. 目录组织规律与命名特征

### 3.1 层级模式

```text
example_data/
  ├─ HSIL/
  │   ├─ 刘从琴/
  │   │   ├─ 1.jpg ... 5.jpg
  │   │   ├─ 刘从琴_检查报告_20240628095200.pdf
  │   │   └─ 着色情况.txt
  │   └─ 蔡夏香/
  │       ├─ 1.jpg ... 5.jpg
  │       ├─ 蔡夏香_检查报告_20210827103915.pdf
  │       └─ 着色情况.txt
  └─ LSIL/
      ├─ 包张红/
      │   ├─ 1.jpg ... 5.jpg
      │   ├─ 包张红_检查报告_20210420163925.pdf
      │   └─ 着色情况.txt
      └─ 包怡青/
          ├─ 1.jpg ... 5.jpg
          ├─ 包怡青_检查报告_20241230090106.pdf
          └─ 着色情况.txt
```

### 3.2 命名规则

- 分类标签使用目录名表达：`HSIL` / `LSIL`。
- 患者目录名使用中文姓名。
- 图像文件统一按序号命名：`1.jpg` ~ `5.jpg`。
- PDF 报告命名格式：`{姓名}_检查报告_{YYYYMMDDHHMMSS}.pdf`。
- 文本描述文件固定命名：`着色情况.txt`。

### 3.3 文件内部结构规律

- **JPG**：分辨率统一为 `696x570`，`24bpp RGB`，`96 DPI`；每位受检者固定 5 张，通常反映同一次阴道镜流程不同阶段（普通视图/醋酸试验后/碘试验后）。
- **TXT**：均为 2 行核心描述，固定字段模式：
  - `阴道镜所见(宫颈): ...`
  - `阴道镜所见(阴道): ...`
- **PDF**：均为单页检查报告，包含人口学信息、TCT/HPV、阴道镜所见、拟诊结果、处理方案与随访日期。

## 5. 数据结构化建模建议（可直接转 JSON）

可将每个受检者统一抽象为：

```json
{
  "label": "HSIL | LSIL",
  "patient_name": "中文姓名",
  "images": [
    "1.jpg",
    "2.jpg",
    "3.jpg",
    "4.jpg",
    "5.jpg"
  ],
  "report_pdf": "姓名_检查报告_YYYYMMDDHHMMSS.pdf",
  "stain_text": "着色情况.txt",
  "stain_findings": {
    "cervix": "阴道镜所见(宫颈)...",
    "vagina": "阴道镜所见(阴道)..."
  }
}
```

该数据天然适用于：

- 病例级分类任务（`HSIL/LSIL`）
- 多模态建模（图像 + 文本 + 报告）
- 规则校验（目录完整性、文件命名一致性）
